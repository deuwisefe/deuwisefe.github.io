讲清楚 this

先抛出两个问题

问题 1：下面代码会打印出什么？

var config = {
  port: ':3389',
  url: 'http://www.deuwise.com' + this.port
};
console.log(config);

问题 2：为什么 jQuery 事件绑定中，回调函数里面的 this 是 dom 元素

$('body').on('click', function() {
  console.log(this);
});

要搞清楚上面三个问题，先要讲一下 this 的定义

this：当前执行代码的环境对象

浏览器有一个默认的 this，即 window 对象，this 出现的地方有两个，函数内和函数外。

函数外

在全局执行环境中（函数外部） this 都指向全局对象。浏览器中的全局对象就是 window

console.log(this); //window

函数内

在函数内部，this 的值取决于函数被调用的方式（箭头函数除外）。

在函数中 this 大概在以下几个场景中出现。

  1. 在对象的方法中
  2. 在普通函数中
  3. 被 call、apply、bind 调用
  4. 在构造函数中
  5. 在箭头函数中

作为对象的方法调用

作为对象的方法调用被调用时，this 指向调用这个方法的对象。即谁调用该方法，this 就指向谁。

var name = 'map2D';
var obj = {
  name: 'map3D',
  f1: function() {
    console.log(this.name);
  }
};
obj.f1(); // map3D

上面代码中，obj 调用了 f1() 方法，所以 f1() 方法中的 this 指向 obj 对象

普通函数中 （隐式绑定）

函数直接调用时，this 指向全局对象，浏览器中是 window，node 中是 global,严格模式下，this 指向 undefined。()

var name = 'map2D';
function f1() {
  var name = 'map3D';
  console.log(this);
  console.log(this.name);
}
f1(); //window，map2D

f1() 相当于是 window.f1() ，所以 this 指向 window

var obj = {
  f1: function() {
    function a() {
      console.log(this);
    }
    a();
  }
};
obj.f1(); // window

同上，a() 相当于 window.a() 所以 this 也是 window

注意，webpack 中默认是严格模式

被 call、apply、bind 调用 (显式绑定)

在 JS 中，函数也是对象，在 Function.prototype 函数的原型下，就有 call()、apply()、bind()这三个字方法，它们可以改变函数的 this 指向

apply()

apply() 方法调用一个具有给定 this 值的函数，以及作为一个数组（或类似数组对象）提供的参数。

换句话说就是会执行该函数，并且可以设置该函数中的 this 。第一个参数就是该函数的 this 指向，第二个参数是执行函数需要传入的参数

function run(a, b, c) {
  console.log(this.name, a, b, c);
}
var obj = { name: 'map' };
run.apply(obj, [1, 2, 3]); // map 1 2 3

求数组中的最大值，Math.max() 是不支持给一个数组参数里的值求值的，在 ES6 中，可以通过的数组的扩展运算符 ... 转换成参数序列来实现。

Math.max(...[12, 23, 4, 55, 11]); // 55

在 ES5 中，可以用 apply 实现

Math.max.apply(null, [12, 23, 4, 55, 11]); // 55

call()

call()方法是 apply() 的语法糖，作用和 apply()一样，唯一的区别就在于 call()接收的是参数列表，而 apply()则接收参数数组。

Math.max.apply(null, 12, 23, 4, 55, 11); // 55

bind()

bind()方法创建一个新的函数，在 bind() 被调用时，这个新函数的 this 被 bind 的第一个参数指定，其余的参数将作为新函数的参数供调用时使用。

区别是call()与apply()会立即执行，而bind()是返回一个新函数，在需要时调用

var page = {
  name: 'hello',
  bindEvent() {
    $('#map').on('click', this.mapClick.bind(this));
  },
  mapClick() {
    console.log(this.name); // hello
  }
};
page.bindEvent();

使用过 jQuery 的都知道，在上面的 jQuery 的事件回调中，this 默认 指向的是当前正在执行事件的元素，也就是#map的 dom 对象，如果要在mapClick()中访问name属性的话，就得把 jQuery 事件回调的 this 指向改变。即this.mapClick.bind(this)

在构造函数中

函数如果作为构造函数使用 new 调用时, this 指向的是新创建的构造函数的实例。

function Foo() {
  console.log(this);
}
var bar = new Foo(); // 输出: Foo 实例，this 就是 bar

在使用new 调用构造函数时，会依次执行下列操作

  1. 创建一个新对象
  2. 构造函数的 prototype 被赋值给这个新对象的 __proto__
  3. 将构造函数中的 this 指向这个新对象
  4. 执行构造函数
  5. 返回新对象

在箭头函数中

箭头函数的 this 比较特殊，是根据声明的地方来决定this的，它是 ES6 中新增的语法

箭头函数的 this 绑定是无法通过 call、apply、bind 被修改的，且因为箭头函数没有构造函数 constructor 所以也不能使用 new 调用，即不能作为构造函数，否则会报错。

使用箭头函数优化上面的 bind 代码

var page = {
  name: 'hello',
  bindEvent() {
    $('#map').on('click', () => this.menuClick());
  },
  mapClick() {
    console.log(this.name); // hello
  }
};
page.bindEvent();

var name = 'hello';
var obj = {
  name: 'world',
  foo: () => {
    console.log(this.name);
  }
};
obj.foo(); // hello

this 优先级

优先级：new 绑定 > 显示绑定 > 隐式绑定 > 默认绑定

问题

最后回到上面的两个问题。

问题 1：下面代码会打印出什么？

var config = {
  port: ':3389',
  url: 'http://www.deuwise.com' + this.port
};
console.log(config);

会打印出 {port: "3389", url: "http://www.deuwise.comundefined"}，上面的代码中的 this.port 就相当于是window.port，window.port 是undefined，字符串和 undefined相加，会将undefined转为 "undefined"字符串，所以最后的结果是"http://www.deuwise.comundefined"

如果想用上面代码中的写法的话，也有办法，可以借助 js 的 get 语法。get 语法将对象属性绑定到查询该属性时将被调用的函数。

var config = {
  port: ':3389',
  get url() {
    return 'http://www.deuwise.com' + this.port;
  }
};
console.log(config.url); //http://www.deuwise.com:3389

问题 2：为什么 jQuery 事件绑定中，回调函数里面的 this 是 dom 元素

$('body').on('click', function() {
  console.log(this);
});

简单模拟实现

function $(ele) {
  return {
    on(event, callback) {
      if (event === 'click') {
        document.querySelector(ele).addEventListener('click', function() {
          callback && callback.call(this);
        });
      }
    }
  };
}

$('#map').on('click', function() {
  console.log(this); //#map节点的dom对象
});

解析：原生的事件绑定，回调函数中的this指向是该 dom 对象，在我们封装的 on 方法中，接受一个回调函数 callback ，在 js 事件回调函数执行完后，执行 callback()，并将 this 用 call(this) 改变指向。
